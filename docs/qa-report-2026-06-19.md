## 4. Bugs and issues found

### Fixed during this QA round

| # | Issue | Severity | Fix |
|---|-------|----------|-----|
| 1 | **Hydration mismatch on `/login`**: base-ui `Input` injected `style={{caret-color:"transparent"}}` on the client only, causing a React hydration warning. | Medium | Replaced base-ui `Input` with plain `<input>` elements on the login page. |
| 2 | **Footer language inconsistency**: Footer links and description were hardcoded in Russian while the rest of the landing page is Turkish. | Medium | Footer now uses Turkish copy from `messages/tr.json` (company, catalog, services, legal columns + description). |
| 3 | **Playwright selector flakiness**: initial card-count selectors matched nested divs. | Low | Added `data-testid` attributes to section components and used direct-child selectors. |

### Open issues for next bug-fixing round

| # | Issue | Severity | Notes |
|---|-------|----------|-------|
| 1 | **Login form is non-functional** | High | There is no auth handler. Submitting the form does nothing and no validation/error messages appear. Needs Supabase env vars + `signInWithPassword` integration. |
| 2 | **Dashboard is not protected** | High | `/dashboard` is reachable without authentication. Once auth is enabled, middleware must redirect unauthenticated users to `/login`. |
| 3 | **Dashboard sidebar not responsive** | Medium | On mobile viewport the sidebar stays open and consumes most of the width. Needs a collapsible drawer / hamburger menu for small screens. |
| 4 | **CTA primary button is not linked** | Medium | "Demo Talep Et" button has no `onClick` or `href`. Should open a form, Cal.com booking, or mailto. |
| 5 | **Hero search widget is decorative** | Medium | City, type, budget dropdowns are static divs. They need real filters and a results page. |
| 6 | **Stats counters start at final value** | Low | Counters do not animate from 0 because of SSR/hydration safety. Acceptable for MVP but could be improved visually. |
| 7 | **Navbar links are mostly anchors** | Low | "Tüm İlanlar", "Hakkımızda", etc. point to `#` or sections. Expected for single-page landing, but catalog page is missing. |
| 8 | **Middleware deprecation warning** | Low | Next.js 16 warns that `middleware.ts` is deprecated in favor of `proxy.ts`. Should migrate once auth is implemented. |
| 9 | **Playwright combined run instability** | Low | `npx playwright test` (both projects) loses dev server between projects. Running per-project is the current workaround. |

## 6. How to run the tests

```bash
cd apps/web

# Run desktop tests
npx playwright test --project=chromium

# Run mobile tests
npx playwright test --project=mobile-chrome

# View report
npx playwright show-report
```

Playwright will automatically start `pnpm dev` on port 3000, run the tests, and save screenshots to `apps/web/qa/`.

## 7. Recommendations for next iteration

1. **Enable Supabase auth** and add real login/logout flows. This unblocks dashboard protection and test data creation.
2. **Build the first CRM data model** (properties, leads, tickets) so the dashboard zeros become real numbers.
3. **Make the dashboard responsive** before showing it on mobile devices.
4. **Add a contact/demo form** and link the CTA button.
5. **Keep Playwright tests green** after every feature by adding new assertions as modules ship.
